2. Hardware Overview

Front

  1. Power LED — Orange while booting → red during startup → green when ready to play
  2. Time display
  3. Status symbols — Network, USB device, playback status

Back

Connectors from left to right:

  1. WLAN antenna connection — Screw on the included 10 cm antenna for improved WiFi reception
  2. AV output — 3.5 mm 4-pin jack
    • Pin 1: GND (Mass)
    • Pin 2: CVBS (composite video)
    • Pin 3: Audio Right
    • Pin 4: Audio Left
  3. Gigabit LAN port — Integrate into an existing network via DHCP or a static IP address
  4. HDMI 2.0 output — Main video output, up to 4K
  5. S/PDIF optical output — Digital audio via TOSLINK cable
  6. Power supply connector — Connect the included 5 V adapter here

Right Side

Connectors from left to right:

  1. USB 3.0 — External devices: mouse, keyboard, presenter, touchscreen, etc.
  2. USB 2.0 — External devices: mouse, keyboard, presenter, touchscreen, etc.
  3. MicroSD slot — Insert MicroSD card with playable files (FAT32, NTFS, exFAT)

3. Getting Started

Connect Outputs

Video

Connect the player to your HDMI display using the HDMI Out port on the back. By default, the player uses automatic HDMI detection — it selects the best resolution reported by the display. If the display sends incorrect information, configure the resolution manually (see section 4.2.1.1).

If no storage medium with playable files is inserted, the home page with version information is displayed. Otherwise, playback begins directly from the MicroSD card or USB memory.

Audio

Three audio outputs are active in parallel — no additional settings needed:

  1. Digital audio via the HDMI output
  2. Digital audio via the S/PDIF optical output
  3. Analogue audio via the 4-pin AV Out jack

Network

A network connection is required for:

  • Displaying external HTML websites
  • UDP remote control
  • Automatic time synchronization via NTP server

To set up the network:

  1. Connect a CAT5 Ethernet cable to the Gigabit LAN port on the back.
  2. The player ships with DHCP enabled. If your network has a DHCP server, no further configuration is needed. The assigned IP address is shown in the setup menu.

To set a static IP address, create or edit a CFPSetup.txt file:

DHCP Off
IP 192.168.0.2
Subnet 255.255.255.0
Gateway 192.168.0.1

Save this file on the SD card or USB stick and insert it into the player. The settings are applied and saved internally.

Starting the Player

Connect the 5 V power adapter. The power LED changes from orange → red (boot screen), then to green when ready to play.

Starting Playback

Copy your media files to the root directory of the SD card or USB memory and insert it into the player (hot-insert during operation is supported).

Playback starts in alphanumeric order (0–9, A–Z). Videos play once; images are displayed for 5 seconds. After all files have played, it loops back to the first.

Special cases:

  • Single video on card → loops seamlessly without black screen
  • Single image on card → displayed endlessly

For custom order or interactive behavior, use a PLAYLIST.txt.

    • 10 bits (HDR)

LAN

4.2.1.3 Synchronization

Important Players to be synchronized MUST be on the same network and use the same UDP port!
Important When using a playlist, the ITEM numbers of master and slave players must match. When the master starts ITEM 5, ITEM 5 also starts on all slaves — regardless of file name.
  • Activation — On / Off
  • Type
    • Master — Controls all slave players. Direct all external controls to the master only. There can only be ONE master per sync group!
    • Slave — Controlled by the master. Any number of slaves is supported.
  • Group — Assign a sync group (Group 1–10). Different groups do not affect each other.

  • Animation — Transition effect between images:
    • Off — No animation
    • Fade — Crossfade
    • Slide — Slide in

  • Animation duration — Duration in milliseconds
Attention If the animation duration overlaps with the display time, image errors may occur.
  • Zoom — Enable / disable pinch-to-zoom on touchscreen for images
  • Swipe — Enable / disable swipe gestures on touchscreen
  • Longclick Timeout — Duration in ms until a long touch is detected
  • DMX — On / Off
  • ArtNet — On / Off
  • Playback mode (without playlist):
    • All files in loop — All files loop in alphanumeric order
    • First file in loop — First file loops; after an external command, returns to the first file

5. Play Files

5.1 Without Playlist

Simplest way to play files:

  1. Copy files to the root directory of the MicroSD card or USB memory
  2. Insert into the player

Playback starts in alphanumeric order (0–9, A–Z). Videos play once; images display for 5 seconds. After all files, it loops.

Example playback order:

  1. 0.jpg
  2. 05_Film5.mp4
  3. 1.mp4
  4. Display1.mov
  5. Finish.jpg

Playback modes (configurable in settings or CFPSetup.txt):

  • All files in loop — All files loop in alphanumeric order
  • First file in loop — First file loops; after an external command, returns to the first file
  • Selected file in loop — Currently selected file loops continuously

5.2 With Playlist (PLAYLIST.txt)

To define a custom order, or to hold a file until user interaction (touch, network command, etc.), create a PLAYLIST.txt file in the root directory of the SD card or USB memory.

Required Parameters

ParameterDescription & Example
[ITEM #] Every entry must start with this line. Replace # with the index (starting at 0). All playback commands refer to this index.
Example: [ITEM 0]
Datei=filename.ext Filename including extension. Case-sensitive. No spaces allowed.
Example: Datei=Film1.mp4

Optional Parameters

ParameterDescription & Example
Displaytime=# Display duration in seconds — valid for images and websites. Default: 5.
Use -1 for endless display until another ITEM is triggered from outside.
Example: Displaytime=6
Succ=# Successor item (by ITEM number).
Example: Succ=1 → ITEM 1 plays next
RandomSucc=#,#,# Random successor chosen from a list of ITEM numbers.
Example: RandomSucc=1,2,4,6,8
Random A random ITEM is chosen as successor.
Loop=# Repeat count. -1 = endless loop; a positive number plays that many times.
Example: Loop=-1 (endless) · Loop=5 (five times)
Volume=# Volume for this item. 0 = 0%, 10 = 100%.
Example: Volume=3 → 30% volume
Interruptible= Whether this item can be interrupted.
TRUE = can be interrupted · FALSE = must play to completion
Orientation= Screen orientation for this item.
Values: Landscape · Portrait · Reverse Landscape · Reverse Portrait
IP x.x.x.x port cmd Send a UDP command when this item starts.
Example: IP 192.168.0.5 4950 LampOn
Sends "LampOn" to IP 192.168.0.5, port 4950.
Relay1=on Relay2=off Switch relays on or off when item starts.
Example: Relay1=on Relay2=off Relay3=on
DMX_Set ch:val,… Set DMX channels when item starts.
Example: DMX_Set 1:255,2:255,3:255,4:255 (channels 1–4 → 255)
DMX_Fade(val,sec) ch,… Fade channels to a value over a duration.
Example: DMX_Fade(255,2) 1,200,300,512
Channels 1, 200, 300, 512 faded to 255 in 2 seconds.
LED1=on LED2=off Control USB IOBoard LED outputs.
Example: LED1=on LED2=off LED3=on

5.5 HTML JavaScript Inject

When Javascript_Inject=On is set in the playlist, a function is injected into the page's JavaScript that allows sending commands back to the player:

MessageToPlayer.send("next");    // jump to next playlist item
MessageToPlayer.send("prev");    // jump to previous playlist item
MessageToPlayer.send("play 3");  // play item with index 3

5.2.7.1 Receive Key Input in the WebView

Key and digital inputs are forwarded to the WebView as a JavaScript function call:

function Key(keyIndex) {
  switch (keyIndex) {
    case 1:
      // Action for key 1
      break;
    case 2:
      // Action for key 2
      break;
  }
}

Key assignment:

  • Key(1) to Key(10) — Board 1 inputs
  • Key(11) to Key(20) — Board 2 inputs
  • … (depending on board addressing)
Reaction to Opening a Contact
function Key_Off(keyIndex) {
  switch (keyIndex) {
    case 1:
      // Action when contact 1 opens
      break;
  }
}

5.2.7.2 Control IOBoard LEDs from JavaScript

callPlayer("LED L1:On,L2:Off,L3:Off");

5.2.7.3 Send RS232 Commands from the WebView

callPlayer("RS232 command to serial port");

Everything after RS232 is sent unchanged via the serial interface.

Receive RS232 Messages in the WebView

The RS232 device must prefix its output with HTML:

HTML RS232_InWebview(message)

Everything after HTML is executed as JavaScript in the WebView:

function RS232_InWebview(message) {
  console.log("Received RS232: " + message);
}

6. Timecode Functions (*.sze File)

File Structure

Each time a video starts, the player checks for an associated .sze file with the exact same name:

Media fileRequired .sze filename
Film1.mp4Film1.mp4.sze

Each command is on its own line. The line starts with @, followed by the timecode, then a > and the command. Accuracy: one frame at 60 fps = 16 ms.

Timecode format: @ HH:MM:ss.ms

Attention Use a period (.) before the milliseconds — not a comma.

All commands except Pause are followed by the > character before the command parameters.

Available Commands

Pause

Pauses playback until resumed manually. No > sign or further parameters.

@ 00:00:01.500

UDP Telegram

Sends a UDP message to any IP address and port.

@ HH:MM:ss.ms > IP PORT telegram
ParameterDescription
IPIP address of the recipient
PORTPort number of the recipient
telegramMessage text to send
 
@ 00:00:01.500 > 192.168.178.50 4950 LampOn

RS232 Telegram

Sends a message via the RS232 serial interface.

@ HH:MM:ss.ms > RS232 telegram

Optional suffixes: [CR] appends carriage return (0x0D) · [LF] appends line feed (0x0A).

@ 00:00:01.500 > RS232 LampOn[CR][LF]

Relay

Open or close relay contacts on the USB relay card.

@ HH:MM:ss.ms > Relay Rn:State[,Rn:State,…]

With one USB card: relays 1–8. With two USB cards: 1–16.
State: On = relay closed · Off = relay open.

@ 00:00:01.500 > Relay R1:On,R2:Off,R7:Off
@ 00:00:07.500 > Relay R1-R7:Off
@ 00:00:10.500 > Relay R3-R6:On

USB IOBoard LEDs

Switch IOBoard output LEDs on or off.

@ HH:MM:ss.ms > LED Ln:State[,Ln:State,…]

With one board: L1–L10. With two boards: L11–L20.

@ 00:00:01.500 > LED L1:On,L2:Off,L3:Off

DMX_Set

Sets specified DMX channels to fixed values immediately.

@ HH:MM:ss.ms > DMX_Set Channel:Value[,Channel:Value,…]

Channels: 1–512 · Values: 0–255

@ 00:00:01.500 > DMX_Set 1:115,2:255,110:255,510:123

DMX_Fade

Fades specified channels to an end value over a duration.

@ HH:MM:ss.ms > DMX_Fade(endValue, fadeDuration) channel1,channel2,…

End value: 0–255 · Fade duration: seconds (any value) · Channels: 1–512

@ 00:00:01.500 > DMX_Fade(255,3) 1,2,3,4

Channels 1–4 faded to 255 in 3 seconds.

7. Synchronization

Synchronize multiple molitor 4K Light players for frame-accurate video wall playback.

Important All players to be synchronized MUST be on the same network and use the same UDP port!
Important When using a playlist, ITEM numbers must match across all players. When the master starts ITEM 5, ITEM 5 also starts on all slaves — regardless of file name.

No limit to the number of synchronized players (apart from IP4 protocol limits). Up to 10 independent sync groups can run on the same network. Synchronization is actively regulated — players remain frame-accurate even during long videos. Devices that join later are immediately synchronized to the current position.

Note All players to be synchronized must be on the same network. See Section 3 — Getting Started for network setup details.

Set Up a Video Wall

Prepare Content

For frame-accurate sync, all videos must have:

  • Exactly the same length
  • Exactly the same frame rate
  • The same, ideally fixed bitrate

Distribute Content to Players

Distribute the content to each player according to your screen layout.

7.1.3.1 Playback Without Playlist

Select Sync Options

  • Activation — Must be set to On on all players in the group
  • Type
    • Master — One player sends sync signals. Direct all external control to this player only.
    • Slave — All other players follow the master
  • Group — All players must be in the same group (Group 1–10)
  • Accuracy
    • Minimal — Sync only at video start
    • Low — Up to 1000 ms deviation accepted
    • Medium — Up to 16 ms deviation accepted
    • High — Up to 3 ms deviation. Note: with audio output on slave players, resynchronization may be audible. Use a lower accuracy if this is a problem.
Important All players in a sync project should use the same accuracy setting.

8.3 Touchscreen / Touch Windows

Playback can be controlled by a connected USB touchscreen. Up to 127 touch windows per item can be defined in the playlist. Each window is valid only for its item's display duration — a new set of windows can be defined for each item.

Touch coordinates are scaled to 1920 × 1080 pixels for all touchscreens (with special exceptions). Origin (0, 0) is at the top-left corner, in Landscape orientation.

Important The 1920×1080 coordinate system only applies in Landscape orientation. See the sections below for other orientations.

8.4 USB IOBoard

Inputs

The USB IOBoard provides 10 inputs (IN1–IN10) for buttons, sensors, or normally-open contacts. A contact is triggered by a short circuit between the IN pin and the corresponding GND pin.

By default, contacts 1–10 trigger ITEM 1–10 respectively (Contact 1 → ITEM 1, Contact 2 → ITEM 2, etc.).

8.5.1.1 Custom Configuration with KeyOff

Override the default mapping by adding Key#= entries to the playlist. This applies only to the item it is in — other items use the default mapping unless they also have custom key entries.

[ITEM 0]
Datei=Film1.mp4
Key1=PLAY 6
Key2=PLAY 5
Key3=PLAY 4
Key4=PLAY 3
Key5=PLAY 2
Key6=PLAY 1

In this example, the key numbering is reversed — pressing contact 1 plays ITEM 6, contact 6 plays ITEM 1.

Respond to Opening a Contact

By default, there is no reaction to opening a contact. Add Key#_Off= entries to handle contact opening:

[ITEM 0]
Datei=Film1.mp4
Key1=PLAY 1
Key1_Off=PLAY 2
Key2=PLAY 3
Key2_Off=PLAY 4

When contact 1 closes → play ITEM 1. When contact 1 opens → play ITEM 2.

Outputs

The USB IOBoard provides 10 transistor-switched outputs.

  • Input voltage: 5–24 V DC
  • Max output current: 1 A per channel
  • Connected to the PS+– connector
Attention Pay attention to polarity when connecting the power supply to the PS+– connector.
If many outputs are active simultaneously, the board may overheat.
Control LED Outputs from Playlist
[ITEM 0]
Datei=Film1.mp4
LED1=on
LED2=off
LED3=on

[ITEM 1]
Datei=Film2.mp4
LED1=off
LED2=on
LED3=off

  • FTP download folder — Subfolder on the FTP server for this player

  • Daily — Every 24 hours

ArtNet/DMX Scenes (Cues)

Create complete scenes where all 512 channels of an ArtNet/DMX universe are set. Scenes are stored as XML files in the config/ subfolder on the player.

To create a scene in the web interface:

  1. Go to the DMX/ArtNet Setup page
  2. Click "New ArtNet/DMX Scene" — a new entry appears in the left-panel scene list
  3. Edit the channel values; enable Live Output to preview in real time via DMX/ArtNet
  4. Click Save scene when satisfied

The scene's index number (shown in the left panel) is used to reference it in the playlist or timecode file.

Reference a scene in the playlist:

[ITEM 0]
Datei=Film1.mp4
ArtNetCue=1

Or trigger at a specific timecode:

@ 00:00:01.500 > ArtNetCue 1
